Talker (LM recives words and outputs codebook speech tokens)

text and audio tokens gets concatenated, and RoPe is different for each one.

uses ECAPA-TDNN

Multi Token Predictor predict 15 codes while main transformer predicts only 1 next.

text+speaker+language+audio, but rope applies different frequencies to them so in self attention they appear as different channels.

[ TEXT INPUT ]       	   [ REFERENCE AUDIO ]
      |                         					|
[ Text Embedder ]       [ Speaker Encoder ] (ECAPA-TDNN)
      |                         					|
      |                 				[ Speaker Vector ]
      |                         					|
      +-------------+-------------------------------+
                	 |
        [ TALKER TRANSFORMER ] <--- [ Previous Audio Codes ]
                 	|
        [   LM HEAD / PROJ   ]
                	 |
        [  PREDICTED CODES   ] (e.g., Codebook 1 Index)
                	 |
        [  CODE PREDICTOR    ] (Predicts Codebooks 2-16)
               		 |
        [   VQ DECODER       ] (The SnakeBeta model )
                	 |
        [   FINAL WAVEOFORM  ]

during fine tuning it is important to fine tune not only talker predictor (LLM) but also sub_talker (if LLM predicts 1 next token sub_talker is like MTP predicts other 15 audio tokeizer tokens, which are more importnat than the LLM as they define small accent in the sound while LLM predicts only a raw sound). MLP layers should have LoRa, they store info about how it should sound.
